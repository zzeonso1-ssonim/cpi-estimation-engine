# 소비자물가 추정 엔진 (CPI 추정 웹앱)

채권전략용 **재현 가능한 CPI 전망 엔진**. PRD v3.1-Opus 기반.
헤드라인 · 근원①(식료품및에너지제외) · 근원②(농산물및석유류제외)를 독립 추정하고,
**재현 게이트(검증 6·7)를 통과한 값만 발행**한다.

## 설계 원칙 (PRD §13)

> 절대 먼저 하면 안 되는 일: ①UI부터 만들기 ②매핑 없이 근원 계산 ③재현게이트 없이 발행 ④헤드라인 YoY만 맞추기

→ 그래서 **계산 코어(`engine/`)가 UI 없이 단독으로 돌고 테스트로 검증**된다. UI(`app.py`)는 얇은 표현층.

## 실행

```bash
pip install -r requirements.txt
python data/seed.py            # DB 생성 + 시드
python tests/test_golden_june.py   # 재현 검증 (ALL PASS 확인)
python -m streamlit run app.py     # 웹앱 실행
```

## 데이터 단계(Tier)

| Tier | 데이터 | 가능 범위 | 상태 |
|---|---|---|---|
| **Tier 1 (현재)** | 월별 3개 총지수 + 10개 버킷 가중치(§4.2) | 지수법·버킷 재정규화·재현게이트·6월 예시 재현 | ✅ 동작 |
| **Tier 2 (적용됨)** | 458 품목별 2022 가중치 + 버킷·근원 포함플래그 | 품목 단위 자동충격·재정규화 | ✅ 동작 |

- **가중치 출처**: 통계청 「2022년 기준 소비자물가지수 가중치 개편 결과」 보도자료 붙임2(458품목).
  `data/extract_weights.py`로 PDF→`item_weights_2022.csv` 추출(458개·합 1000.0 검증), `data/map_items.py`로 버킷·근원 매핑.
- **혼재버킷 잔차(투명 노출)**: 일부 서비스 품목 공공/개인 경계 근사 → 버킷합과 공표값의 잔차를
  seed 시 진단 출력(§1.3). 단, 개별 품목 가중치 자체는 보도자료값으로 정확하다.
- **KOSIS 월별 지수 자동수집**: `.env`에 `KOSIS_API_KEY` 설정 → `python kosis_fetch.py` 또는
  앱 사이드바 「🔄 KOSIS 월별 지수 동기화」. 표: 헤드라인 `DT_1J22003`·근원① `DT_1J22009`·근원② `DT_1J22007`.

## 전망 모드

- **direct**: 공표 근원 총지수 + 근원 MoM 가정을 직접 지수법에 투입. §11 6월 예시 경로.
- **bottom-up**: 버킷별 MoM 가정 → 재정규화(§5.3)로 근원 MoM 합성. 검증6(Σ=1000) 자동 확인.
