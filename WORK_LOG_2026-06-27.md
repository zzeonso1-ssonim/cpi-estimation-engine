# CPI 추정엔진 작업 기록

- 작업일: 2026-06-27
- 대상 프로젝트: CPI 추정엔진
- 요청 사항:
  1. 전망모드의 기본 화면을 Bottom-up으로 변경
  2. 참가격 생필품 정보를 활용해 전망 정합도 보강

## 1. 전망모드 기본 화면 변경

- `app.py`의 전망모드 선택 순서를 조정했다.
- 앱을 처음 열었을 때 `Bottom-up` 전망 화면이 기본으로 표시된다.
- 기존의 다른 전망 방식은 선택지로 유지했다.

## 2. 참가격 생필품 정보 연동

- 연동 사이트: https://www.price.go.kr/tprice/index.do#
- `web_update.py`에 참가격 페이지 수집 및 해석 로직을 추가했다.
- 생필품별 현재 가격과 이전 가격을 추출해 품목별 가격 변동률을 계산한다.
- 수집 품목을 CPI 입력 구조에 맞춰 다음 네 개 버킷으로 매핑한다.
  - 가공식품
  - 축산물
  - 농산물
  - 공업제품
- 최근 2주 기준 중앙값을 대표 신호로 사용한다.
- 이상치 영향을 줄이기 위해 적용값을 `-3% ~ +3%` 범위로 제한한다.
- 표본이 2개 미만인 버킷은 자동 적용 대상에서 제외한다.

## 3. 화면 구성

- `app.py`에 `참가격 생필품 새로고침` 기능을 추가했다.
- 수집 결과를 표로 확인할 수 있도록 구성했다.
- 참가격 신호를 Bottom-up 전망 입력값의 기본값과 연결했다.
- 사용자가 화면에서 값을 검토하고 직접 수정할 수 있는 기존 흐름은 유지했다.

## 4. 분류 정확도 보완

- 짧은 문자열이 상품명 일부와 우연히 일치하는 문제를 수정했다.
- 예: 브랜드명 `풀무원`의 `무`가 농산물로 잘못 분류되는 사례를 방지했다.
- 분류 우선순위를 `가공식품 -> 축산물 -> 공업제품 -> 농산물` 순으로 조정하고, 더 구체적인 키워드를 사용하도록 보완했다.

## 9. 실행 방법

프로젝트 폴더에서 다음 명령으로 실행한다.

```powershell
& 'C:\Users\LENOVO\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe' -m streamlit run app.py --server.address 127.0.0.1 --server.port 8501 --server.headless true --browser.gatherUsageStats false
```

실행 후 브라우저에서 아래 주소로 접속한다.

```text
http://127.0.0.1:8501
```

## 10. 참고 사항

- 참가격 연동은 공식 CPI 통계를 대체하지 않고 단기 방향성을 보강하는 보조 신호로 사용한다.
- 사이트 HTML 구조가 바뀌면 파서 보완이 필요할 수 있다.
- 현재 백그라운드 서버는 실행 중이 아니며, 위 명령으로 다시 시작할 수 있다.
